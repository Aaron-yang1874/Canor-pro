# 安全策略

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| 2.0.x | ✅ 活跃开发 |
| 1.x.x | ❌ 已弃用 |

## 报告安全漏洞

我们非常重视安全漏洞。如果您发现了安全漏洞，请通过以下方式报告：

**不要**通过 GitHub Issues 报告安全问题！

请发送邮件至：`security@canor.example.com`

请在邮件中包含以下信息：

1. 安全漏洞的描述
2. 复现步骤
3. 受影响的版本范围
4. 可能的修复建议（如果有）

## 响应时间

- 初步响应：24 小时内
- 问题评估：72 小时内
- 修复发布：视严重程度而定

## 安全更新

安全更新将通过以下渠道发布：

- GitHub Security Advisories
- Release Notes
- 邮件通知（对于主要版本）

## 安全最佳实践

使用 Canor 时，请遵循以下安全最佳实践：

### 密钥管理

- 始终使用环境变量存储敏感信息
- 定期轮换 API 密钥
- 不要将凭据提交到版本控制

### 生产部署

- 启用 HTTPS
- 配置适当的 CORS 策略
- 启用安全响应头
- 定期更新依赖

## 安全相关配置

```env
# 生产环境必须配置
JWT_SECRET=<your-secure-jwt-secret>
ENCRYPTION_MASTER_KEY=<your-master-key>
RATE_LIMIT_PER_MINUTE=100
```

## 依赖安全

我们使用以下工具监控依赖安全：

- npm audit
- Dependabot 自动更新

## 漏洞赏金

我们目前没有运行漏洞赏金计划。对于重大安全发现，我们会在 릴리스说明中致谢。

感谢您帮助保护 Canor 用户的安全！
